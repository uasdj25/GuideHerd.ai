# GuideHerd Roadmap

Direction through October 2026. Items in **Martinson & Beason Demo** reflect
current work. Items further down are **planned or exploratory, not commitments**,
and are expected to move as we learn.

## Martinson & Beason Demo

The first end-to-end demonstration of GuideHerd-branded scheduling.

- End-to-end voice scheduling
- Outlook integration *(current calendar provider; replaceable)*
- Confirmation workflow
- GuideHerd-branded demo experience

## Pilot Deployment *(planned)*

First real receptionist-facing deployment.

- Receptionist Portal MVP
- Context Handoff API
- Multi-Attorney Scheduling
- Telephony Integration Research
- Live Call Transfer Prototype
- White-Labeled Notifications

## GuideHerd v1.0 *(future)*

Hardening scheduling into a supportable product with self-service configuration.

- Lex Scheduling Orchestration
- Booking Summary Generation
- Customer Configuration
- Customer Administration Portal
- Monitoring Dashboard
- System Health Monitoring
- Customer Onboarding Documentation

---

As a rule, every recurring manual configuration step done during the demo and
pilot becomes an Administration Portal capability in v1.0. See
[ARCHITECTURE.md](../ARCHITECTURE.md) for the principles behind these choices.
